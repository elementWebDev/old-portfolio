Section

While a div has no semantic meaning, a section element does, and it's used for grouping together thematically related content.

“Is all of the content related?”

If you answered yes to that question, then a section element might be appropriate.

Before

    <div class="section">
      <h2>The Gallery</h2>
      <!--...-->
    </div>

HTML5


    <section>
      <h2>The Gallery</h2>
      <!--...-->
    </section>

Article

The article element represents a complete, or self-contained, composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication.

Technically just independently reusable or distributable. It's just something we can use in syndication. Essentially, an article element is just another type of section. It's used for self-contained related content.

Determining if a particular piece of content is “self-contained”:

Some uses for the article tag:

  • a blog post
  • a news story
  • a comment or post
  • a review

For example: each item in the sample Gallery would be marked up as an article.


        <article>
          <!-- ... -->
        </article>

Main Content

Figure / Figcaption

Figure

The figure element represents a unit of content, optionally with a caption, that is self-contained, that is typically referenced as a single unit from the main flow of the document without affecting the document's meaning.


        <figure>
          <img src="image.jpg" alt="My Picture" />
        </figure>

Figcaption

The figcaption element represents a caption or legend for a figure.


      <figure>
        <img src="image.jpg" alt="My Picture" />
        <figcaption>This is a caption for the picture.</figcaption>
      </figure>

So for our existing figure element, we add a figcaption element that contains a caption for the picture. Make sure to put the image and figcaption inside the figure element.

Time

The time element represents either a time on a 24 hour clock, or a precise date in the proleptic Gregorian calendar, optionally with a time and time-zone offset.

Example usage of the time tag:


      <time>2013-09-16</time>

Say we want to display time as “mm/dd/yyyy”:


      <time>09/16/2013</time>

But we actually want to use a different format for our date- the one one shown above- so we're going to put that date format below within the content for our time tag. We need to make an important change in order to use that date format


      <time datetime="2013-09-16">09/16/2013</time>

Without the datetime attribute, content must be a valid date, time, or precise datetime.
